# i18n (EN / PT-PT) Implementation Design

**Goal:** Add full English and Portuguese (PT-PT) support вЂ” URL-based routing, YAML-driven UI translations, and bilingual blog posts with graceful fallback for untranslated content.

**Architecture:** Custom lightweight i18n service (no external i18n library). YAML source files compiled to JSON at build time. Angular Router restructured around a `/:lang` prefix segment. Blog content split into per-language subfolders.

**Tech Stack:** Angular 21 standalone, `js-yaml` (build script only), Angular Router, Angular Signals, `HttpClient`.

---

## 1. URL Structure

Every route gains a language prefix as the first segment:

```
/                         в†’ detect lang в†’ redirect to /pt/ or /en/
/pt/                      в†’ home (PT)
/en/                      в†’ home (EN)
/pt/blog/:slug            в†’ blog post (PT, or redirect to /en/blog/:slug if no PT version)
/en/blog/:slug            в†’ blog post (EN)
/pt/publications          в†’ publications (PT)
/en/publications          в†’ publications (EN)
```

Invalid language segments (anything other than `en` or `pt`) redirect to `/pt/`.

## 2. Language Detection and Persistence

A `LanguageGuard` (functional guard) handles the root `/` route:

1. Read `localStorage.getItem('lang')` вЂ” if `'en'` or `'pt'`, redirect to `/{lang}/`.
2. Else read `navigator.language` вЂ” if starts with `'pt'`, use `'pt'`, else `'en'`.
3. Redirect to `/{lang}/`.

`localStorage` is written by `I18nService.load()` on every language load. Manual language switch via the header switcher updates `localStorage` immediately before navigating.

## 3. Router Structure

```
/                          в†’ LanguageGuard в†’ redirect
/:lang                     в†’ LangShellComponent (validates lang, loads translations via resolver)
  ''                       в†’ HomePageComponent
  blog/:slug               в†’ BlogPostPageComponent
  publications             в†’ PublicationsPageComponent
**                         в†’ redirect to /pt/
```

`LangShellComponent` is a minimal wrapper with `<router-outlet>`. It validates that `:lang` is `'en'` or `'pt'`; invalid values redirect to `/pt/`.

A route resolver (`I18nResolver`) attached to the `/:lang` route calls `i18nService.load(lang)` and returns an `Observable<void>`. The router waits for resolution before rendering children вЂ” no flash of untranslated text.

## 5. Build Pipeline

**Script:** `scripts/generate-i18n.js` (ESM, alongside existing `generate-postcard-manifest.js`)

- Reads `src/i18n/en.yaml` and `src/i18n/pt.yaml` using `js-yaml`
- Flattens nested keys to dot-notation: `{ "hero.title": "...", "nav.blog": "..." }`
- Writes `public/i18n/en.json` and `public/i18n/pt.json`

**package.json hooks** (extend existing `prestart`/`prebuild`):

"prebuild": "node scripts/generate-postcard-manifest.js && node scripts/generate-i18n.js"

## 6. I18nService

**Location:** `src/app/features/i18n/i18n.service.ts`

```typescript
@Injectable({ providedIn: 'root' })
export class I18nService {
  private readonly http = inject(HttpClient);
  readonly lang = signal<'en' | 'pt'>('pt');
  private readonly translations = signal<Record<string, string>>({});

  load(lang: 'en' | 'pt'): Observable<void> {
    return this.http.get<Record<string, string>>(`/i18n/${lang}.json`).pipe(
      tap(data => {
        this.translations.set(data);
        this.lang.set(lang);
        localStorage.setItem('lang', lang);
      }),
      map(() => void 0),
    );
  }

  t(key: string): string {
    return this.translations()[key] ?? key;
  }
}
```

The `translations` signal means Angular's signal-based change detection propagates language switches to all `TranslatePipe` consumers automatically.

## 7. TranslatePipe

**Location:** `src/app/features/i18n/translate.pipe.ts`

```typescript
@Pipe({ name: 'translate', standalone: true, pure: false })
export class TranslatePipe implements PipeTransform {
  private readonly i18n = inject(I18nService);
  transform(key: string): string {
    return this.i18n.t(key);
  }
}
```

`pure: false` so the pipe re-evaluates when the language changes. Usage in templates:

```html
<h1>{{ 'hero.title' | translate }}</h1>
<a>{{ 'nav.blog' | translate }}</a>
```

All components that use `TranslatePipe` import it in their `imports` array.

## 8. Blog Bilingual Structure

**File layout:**

```
public/content/blog/
  index.json                                              в†ђ single master index
  en/
    what-a-microfluidic-chip-teaches-about-storytelling.md
    from-lab-notebook-to-visual-explainer.md
    four-rules-i-use-when-explaining-oncology.md
    behind-the-scenes-pint-of-science-guimaraes.md
    monthly-reflection-template.md
  pt/
    (empty to start вЂ” .md files added as posts are written)
```

**Single master `index.json`** lists every post regardless of language, with a `languages` array:

```json
{
  "posts": [
    {
      "slug": "what-a-microfluidic-chip-teaches-about-storytelling",
      "title": "What a Microfluidic Chip Teaches About Storytelling",
      "date": "2026-05-10",
      "excerpt": "...",
      "languages": ["en"]
    },
    {
      "slug": "um-post-so-em-portugues",
      "title": "Um Post SГі em PortuguГЄs",
      "date": "2026-05-20",
      "excerpt": "...",
      "languages": ["pt"]
    },
    {
      "slug": "bilingual-post",
      "title": "Bilingual Post",
      "date": "2026-05-15",
      "excerpt": "...",
      "languages": ["en", "pt"]
    }
  ]
}
```

**`BlogPostMeta` model update:**

```typescript
export interface BlogPostMeta {
  readonly slug: string;
  readonly title: string;
  readonly date: string;
  readonly excerpt: string;
  readonly cover?: string;
  readonly languages: readonly ('en' | 'pt')[];
}
```

**Runtime behavior:**

`BlogService.getPosts(lang)` reads the single master index. It returns all posts. For each post, if `lang` is not in `post.languages`, the post is marked `unavailable` (computed at runtime, not stored in JSON).

Blog card rendering:
- `languages: ['en']` shown in PT listing в†’ badge using `blog.only_in_en` key ("Only available in English"), link navigates to `/en/blog/:slug`
- `languages: ['pt']` shown in EN listing в†’ badge using `blog.only_in_pt` key ("Only available in Portuguese"), link navigates to `/pt/blog/:slug`
- `languages: ['en', 'pt']` в†’ no badge, link uses current lang

Direct navigation to `/{lang}/blog/:slug` when no matching `.md` file exists в†’ `BlogService` catches the 404 and returns `null` в†’ page component redirects to the other language's URL.

`BlogService` fetches markdown from `/content/blog/{lang}/{slug}.md`.

## 9. Language Switcher UI

Located in the site header (`app.html`), always visible.

```html
<div class="lang-switcher" aria-label="Language">
  <a [routerLink]="langPath('pt')" [class.active]="lang() === 'pt'">PT</a>
  <a [routerLink]="langPath('en')" [class.active]="lang() === 'en'">EN</a>
</div>
```

`AppComponent` injects `I18nService` and `Router`. `langPath(target)` swaps the first URL segment: `/pt/blog/slug` в†’ `/en/blog/slug`. `lang()` reads `I18nService.lang` signal for active state.
